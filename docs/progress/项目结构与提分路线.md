## 4. 提分优先级

### P0：先修复复现性

1. 去掉脚本运行时自动 `pip install` 的逻辑，改成提前安装依赖或 graceful skip。
2. 确认 `requirements.txt` 包含当前主 Pipeline 需要的库：`optuna`, `shap`, `xgboost`, `lightgbm`, `catboost`。
3. 给主脚本加一个快速模式：低 trials、低 folds，用于验证代码无误。

收益：不直接涨分，但这是继续实验的地基。

### P1：统一“文档说有”和“脚本真用”的特征

重点核对这些特征是否真正进入当前 `FEAT_COLS`：

1. `CabinAgreementScore`, `CabinMean`
2. `Surname_tfidf_0` ~ `Surname_tfidf_4`
3. WOE 特征
4. `TE_Surname`
5. `GroupMean`, `GroupAgreementScore` 是否应继续排除

收益预期：如果发现有强特征生成了但没进模型，可能带来 0.002 到 0.008 的 LB 提升。

### P2：增强模型调参

1. LightGBM trials 从 80 提到 300+。
2. 单独给 XGBoost 和 CatBoost 做 Optuna，而不是只继承 LGB 参数。
3. 用相同 folds 保存每个模型 OOF，避免不同实验之间不可比。

收益预期：0.003 到 0.010。

### P3：多 run 稳定集成

1. 固定特征后跑多 seed、多 fold。
2. 保存每个 run 的 OOF/test probability。
3. 用 OOF 学习 blend 权重，而不是只融合最终 csv。

收益预期：0.002 到 0.006，主要提升稳定性。

### P4：规则与阈值微调

当前 `CryoSleep+NoSpend` 已有效。下一步只建议做“OOF 可验证”的规则：

1. 对 `CabinNum` 高一致性样本做 rule lock。
2. 对 `Surname` 高一致性样本做 rule lock。
3. 在未被 hard rule 覆盖的 63% 样本上单独搜阈值。

收益预期：0.002 到 0.008，但必须谨慎防止 LB 过拟合。

### P5：重新确认数据 split

如果能恢复 train/test `GroupId` 有重叠的数据版本，Group Label Propagation 可能是最大增益来源。当前文档估计这一路能带来 0.015 到 0.020，但现在的数据约束下它不可用。

## 5. 下一步推荐执行顺序

1. 修复依赖与一键复现。
2. 生成当前 `FEAT_COLS` 清单，与文档特征表逐项对齐。
3. 把生成但未使用的强特征纳入模型，做一次 OOF 对比。
4. 单独调 XGBoost / CatBoost。
5. 做多 seed OOF 概率级融合。
6. 最后才做规则搜索和 csv blend。
